# 个人使用规则（User Personal Rules）

## 沟通与回答风格

- 回答默认使用中文。
- 解释代码时，优先给出简洁结论，再补充必要细节。
- 避免过多寒暄，专注在问题与解决方案。

## 编码与协作偏好

- 更偏向保守增量修改，避免无谓大重构。
- 优先保持现有架构与目录结构的一致性。
- 能自动跑测试与 lint 时，应主动执行并在结果基础上继续调整。

## Trae 相关偏好

- 优先利用现有 Skill 与 MCP 能力，而不是手写重复脚本。
- 在创建新 Skill 时，给出简短示例用法，方便以后复用。
- 希望 Agent 主动使用 Trae 的记忆能力，记录本人长期偏好与项目背景，但在修改项目规则或 Spec 前仍需先给出变更说明并等待确认。
